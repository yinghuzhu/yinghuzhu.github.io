Biometric-Based Phone-Less Payment Product

发表于 2025-04-07

1. Project Name

Biometric-Based Phone-Less Payment Product for Saudi Arabia

2. Executive Summary

This project aims to provide an innovative, convenient, and secure payment solution for users in Saudi Arabia, allowing them to make payments without a phone using biometric features (such as palm vein scanning). The product will integrate Tap Company’s payment API and biometric capture technology from a partner vendor to deliver a new payment experience for users and an efficient and secure method of receiving payments for merchants.

3. Project Goals

  • To provide Saudi Arabian users with a convenient way to make payments without a phone.
  • To enhance payment security and reduce the risk of fraud.
  • To simplify the payment process and improve transaction efficiency.
  • To offer merchants a modern method of receiving payments, enhancing the user experience.
  • To comply with local payment regulatory requirements in Saudi Arabia.

4. Target Audience

Consumers within Saudi Arabia, particularly those who value convenience, security, and are open to trying new technologies.

5. Product Feature List

5.1 User Features:

  • User account registration (via App).
  • Biometric feature (e.g., palm vein) capture and linking to the user account (via App).
  • Identity verification (may be required, via App).
  • User agreement and privacy policy signing (via App).
  • Initiating payments by scanning biometric features at participating merchants.
  • Payment success notification (via biometric capture device).
  • Transaction history inquiry (via App).
  • User account security management (e.g., password change, via App).

5.2 Merchant Features:

  • Receiving biometric payments from users.
  • Viewing transaction records (via Admin Backend).
  • Checking settlement information (via Tap’s merchant portal).
  • Managing merchant information (via Admin Backend).

5.3 System Management Features (for operations and maintenance personnel):

  • User account management.
  • Transaction management.
  • Device management.
  • System configuration management.
  • Reporting and statistics functions.

6. System Architecture (High-Level Overview)

This project involves the collaborative work of the following main components:

  • Biometric Capture System: Provided by a partner vendor, responsible for capturing users’ biometric information and performing initial processing and secure transmission.
  • User App: The main entry point for user registration, account management, transaction history viewing, etc.
  • Backend System: Our core business logic processing center, responsible for user management, biometric data management, interaction with Tap API, transaction processing, security control, merchant management, etc.
  • Tap Payment API: Used to process actual payment transactions, including fund deduction and settlement.
  • Admin Backend (Web): Used by merchants and system administrators to view transaction records, manage accounts, generate reports, etc.

8. Project Preparation Work

  1. Establish partnership with Tap Company.
  2. Establish partnership with the biometric capture vendor.
  3. Detailed requirements analysis and system design.
  4. Team assembly.
  5. Development environment setup.
  6. Develop a detailed project plan and timeline.
  7. Legal and compliance preparation.

9. Integration with Tap Company

Our backend system will interact with Tap Company’s API interface, mainly including:

  • Initiating payment requests.
  • Receiving payment results.
  • Checking transaction status (may be required).
  • Processing refunds (if supported).
  • Using Tap’s security measures.

10. Integration with Biometric Capture Vendor (Specific details to be determined after discussion)

Our system needs to integrate with the biometric capture system provided by the partner vendor, mainly including:

  • Biometric data capture.
  • Biometric data transmission.
  • Biometric data verification.
  • Secure communication.

11. Security Considerations

  • Data encryption.
  • Authentication and authorization.
  • API security.
  • Prevention of man-in-the-middle attacks.
  • Regular security audits.
  • Compliance with PCI DSS standards (if applicable).

12. Compliance with Saudi Regulations

  • Need to comply with payment regulations in Saudi Arabia.
  • Need to comply with data privacy protection laws in Saudi Arabia.
  • May need to communicate with local regulatory authorities to ensure product compliance.

13. Merchant Collaboration Process

  1. Merchant registration and agreement signing.
  2. Obtain Tap Merchant ID and Authorization.
  3. Associate merchant information in our system.
  4. Deploy biometric payment devices at merchant locations.
  5. Training and support for merchants.
  6. Merchants can view transaction records and manage store information through the Admin Backend.

Link Bank Card

sequenceDiagram
    participant User
    participant App
    participant Backend
    participant Tap as Tap Payment API

    User->>App: Initiate Card Linking
    App->>Backend: Request Card Linking Initiation (User ID)
    Backend->>App: Instruct App to Present Tap's Secure Form
    App->>User: Display Tap's Secure Card Input Form
    User->>App: Enter Bank Card Details
    App->>Tap: Send Card Details Securely (via SDK or Payment Page)
    Tap-->>App: Return Card Token
    App->>Backend: Send Card Token and User ID
    Backend->>Database: Associate Card Token with User Account
    Database-->>Backend: Return Association Result
    Backend-->>App: Return Card Linking Success Confirmation
    App->>User: Display "Card Linked Successfully"

Payment

sequenceDiagram
    participant User
    participant Kafi
    participant Backend
    participant Tap as Tap Payment API

    User->>Kafi: Initiate Payment Request
    Kafi->>Backend: Send Payment Request (Device ID, May Contain Initial User Identification)
    Backend->>Kafi: Request Biometric Verification
    Kafi->>User: Prompt User for Biometric Verification
    User->>Kafi: Complete Biometric Verification
    Kafi->>Backend: Send Biometric Data
    Backend->>Database: Query User Biometric Information
    Database-->>Backend: Return User Biometric Information
    alt Verification Successful
        Backend->>Database: Retrieve User's Card Token
        Database-->>Backend: Return Card Token
        Backend->>Tap: Initiate Payment Request (Amount, Card Token, Merchant ID, etc.)
        Tap-->>Backend: Return Payment Result
        Backend->>Kafi: Return Payment Success Information
        Kafi->>User: Display Payment Success
    else Verification Failed
        Backend->>Kafi: Return Payment Failure Information
        Kafi->>User: Display Payment Failure
    end

sequenceDiagram
    participant User as 用户
    participant MerchantApp as 商户应用
    participant MerchantServer as 商户服务器
    participant WeChatPayServer as 微信支付服务器

    %% 1. 用户发起支付请求
    User->>+MerchantApp: 点击支付按钮
    MerchantApp->>+MerchantServer: 请求生成支付二维码

    %% 2. 商户服务器调用微信统一下单API
    MerchantServer->>+WeChatPayServer: POST /v3/pay/transactions/native
        Note right of WeChatPayServer: 请求参数：
        Note right of WeChatPayServer: appid, mchid, description, out_trade_no,
        Note right of WeChatPayServer: notify_url, amount, payer
    WeChatPayServer-->>-MerchantServer: 返回支付二维码信息
        Note left of MerchantServer: 响应包含：
        Note left of MerchantServer: code_url（二维码链接）、
        Note left of MerchantServer: out_trade_no、
        Note left of MerchantServer: transaction_id

    %% 3. 商户生成二维码并展示给用户
    MerchantServer-->>-MerchantApp: 返回 code_url
    MerchantApp->>+QRCodeGenerator: 生成二维码（code_url）
    QRCodeGenerator-->>-MerchantApp: 返回二维码图片
    MerchantApp-->>-User: 展示二维码

    %% 4. 用户扫码支付
    User->>+WeChatPayScanner: 扫描二维码
    WeChatPayScanner->>+WeChatPayServer: 发起支付请求
    WeChatPayServer-->>-WeChatPayScanner: 支付处理中...

    %% 5. 微信支付服务器异步通知商户
    WeChatPayServer->>+MerchantServer: POST /notify_url (支付结果)
        Note right of MerchantServer: 通知内容：
        Note right of MerchantServer: out_trade_no, transaction_id,
        Note right of MerchantServer: trade_state（SUCCESS/FAIL）
    MerchantServer-->>-WeChatPayServer: 返回 success（必须）

    %% 6. 商户处理业务逻辑（如发货）
    MerchantServer->>+Database: 更新订单状态为已支付
    MerchantServer->>+LogisticsSystem: 触发发货流程
